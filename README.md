# Math Expression Compiler

This project is a small-scale compiler and virtual machine designed to parse mathematical expressions, construct an Abstract Syntax Tree (AST), and compile them into efficient bytecode for execution.

## Project Structure

### Lexical Analysis (Lexer & Tokenizer)

This layer is responsible for scanning the raw input string and breaking it into meaningful units.

* **`lexer.h / lexer.cpp`**
* Reads the input string as an `istream`.
* Provides low-level stream navigation using `peek()` and `advance()` methods.

* **`tokenizer.h / tokenizer.cpp`**
* Utilizes the Lexer to group characters into **Tokens** (e.g., Numbers, Identifiers, Operators).
* Handles whitespace skipping and multi-character token recognition (like floating-point numbers or variable names).

### Abstract Syntax Tree (AST)

Defines the hierarchical structure of the parsed expression.

* **`ast.h`**
* `enum class OpCode`: Defines the set of operations supported by the Virtual Machine (e.g., ADD, SUB, MUL, DIV, UNARY, LOAD).
* `struct Instruction`: The core bytecode unit containing an `OpCode`, operand addresses (`left`, `right`), and a destination index (`dest`).
* **`class ASTNode`**: An abstract base class providing two vital interfaces:
* `print()`: Recursively displays the AST structure in the console with indentation for visual debugging.
* `compile()`: A recursive method (Post-order traversal) that flattens the tree into a vector of `Instruction` objects.

* **Concrete Nodes**:
* `NumberNode`: Represents constant numeric values.
* `VariableNode`: Represents variables with references to the `SymbolTable`.
* `BinaryOpNode`: Handles operations with two operands (+, -, *, /).
* `UnaryOpNode`: Handles single-operand operations (e.g., negation).

### Parser

* **`parser.h / parser.cpp`**
* Converts the stream of tokens into a structured **AST**.
* Implements operator precedence and handles parentheses to ensure correct mathematical evaluation order.

### Executable Part

* **`calc.h / calc.cpp`**
* Contains the `disassemble()` function which displays the generated bytecode.
* Contains the `runVM()` function which executes the generated bytecode.
* Uses a linear memory buffer (`std::vector<double>`) to store intermediate results, making execution significantly faster than direct tree evaluation.

## Execution Pipeline

1. **Source Code** (String) → **Lexer/Tokenizer** (Tokens)
2. **Tokens** → **Parser** (AST Tree Construction)
3. **AST Tree** → **Compile** (Bytecode Generation)
4. **Bytecode** → **runVM** (Final Numeric Result)
